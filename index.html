Session Pad Grid

Advanced options

MIDI: not connected · G = mute column · H = hold stop col, then tap clip or G in that column (Session + Modern).

Press a pad after connect — last MIDI message appears here (decoded as Session pads only). Clip pads use Note On (raw 90 …); Control Change (b0 …) is ignored for clips. On Mini MK3, the right column (scene launches, logo, stop strip) sends CC on numbers that overlap the Session note map — when known, the grey line names that control. With row/column flip on, pack slot … after flip is the internal channel address; the lit grid cell matches the Session key in the same line.

Guide

Packs & audio

Uses the same pack.json + sample layout as Launchpad Arcade (Novation).

  • Local soundlib — /soundlib/<slug>/pack.json (slugs in js/config.js).
  • Novation (proxy) — /novation/ via python3 server.py (same-origin, avoids CORS).
  • Custom URL — one pack.json URL, or a catalog.json manifest ({ "packs": [ { "slug", "title" } ] }) listing many packs; click Load. A catalog enables the Sample set dropdown for switching. On GitHub Pages: catalog.github-pages.json — repo comboshreddies/session-pad-grid. See README.md.
  • Click Enable audio once so Web Audio and bar-sync can run.

Pad names & grid

  • Codes are digit then letter: 1A = column 1, row A (not “A1” on silkscreen).
  • Rows A–F = clips; rows G / H = strip controls (like hardware).
  • Packs with more than six clip rows: grid shows six at a time; on Mini MK3 use DAW input ▲ / ▼ to scroll.
  • Clip rows follow pack JSON order (A at top). Use Flip (advanced, below grid) if left/right or up/down is mirrored vs your Launchpad.

Row G (mute) & row H (stop)

  • G1–G7 — hold to mute column (release unmutes on hardware note off; web uses hold). Playing clips turn white while muted.
  • volume (8G) — hold or click to lock the clip volume menu; row H shows 1/8…8/8 (8H = max).
  • Right column row 3 (web row C, orange pan / Mini MK3 CC 69) — hold for stereo pan: select clips 1A…8F; row G = left L1…L8, row H = right R1…R8 (L+R=8, e.g. G8→R0, H8→L0, G4→R4).
  • H1–H7 — hold stop col (orange): nothing stops until you tap a clip in that column or press G (stop col, white) to stop the whole column.
  • 8H — 8/8 max volume while the volume menu is active; R8 while pan is active; otherwise hold or click to lock clock tick sync and pick ticks with 8A…8F.
  • 1G…7G — one purple pad moves as a visual bar clock (8 steps per loop).

Clip colours

  • Blue / green — idle (odd / even columns).
  • Orange — armed (waiting for next bar/beat when syncTo is set).
  • Yellow — playing; red — playing but stop queued on next grid step.
  • Clock sync menu — purple ticks on row G between bar-sync snaps (0 = immediate, then 1 / 2 / 4 / 8 / 16 ticks).

MIDI & Launchpad mode

  • Use Session on the unit so pads match 1A…8H on this grid.
  • User / Drum / Keys send different note numbers — highlights and debug text can be wrong (accidental matches). Example: corner as 1A in User may decode as 1E in Session.
  • Two USB ports are common: … MIDI (User/Drum/Keys) vs … DAW (Session). Pick the input for the mode you use.
  • Modern map = Arcade pack layout; Classic = older Session note table. DAW inputs are decoded with Classic while the pack map stays Modern.
  • Optional DAW Session SysEx — browser drives Session LEDs (like Arcade). If the hardware Session key stops working, click Release DAW mode or replug USB.
  • Connect MIDI may ask for SysEx permission when the checkbox is enabled.

MIDI debug line

After connect, the grey line shows the last message as a Session pad. Clips use Note On (90 …); CC (b0 …) is ignored for clips. Mini MK3 side buttons may send CC on numbers that overlap the note map.
